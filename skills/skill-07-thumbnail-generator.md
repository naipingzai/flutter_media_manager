# Skill-07: 缩略图生成器规范

## 前置依赖
skill-00

## 目标
定义图片缩略图生成、视频帧提取、缓存策略的完整规范。

---

## 1. 缩略图生成规格

| 规格 | 值 | 说明 |
|------|---|------|
| 目标尺寸 | 256 × 256 像素 | 正方形 |
| 缩放模式 | 居中裁切 | 保持宽高比，裁切到正方形 |
| 输出格式 | JPEG | 压缩质量 80% |
| 存储位置 | 应用私有目录/thumbnails/ | 独立子目录 |
| 文件命名 | {mediaId}.jpg | 与媒体 ID 关联 |

---

## 2. 图片缩略图生成流程

```
输入：源图片文件路径
    │
    ├─ 1. 读取图片尺寸（不加载全部像素）
    │
    ├─ 2. 计算采样率（inSampleSize）
    │     目标：加载后的图片 ≈ 512 × 512（比最终输出大 2 倍，保证质量）
    │
    ├─ 3. 按采样率解码图片
    │
    ├─ 4. 居中裁切为正方形
    │     - 宽 > 高：裁切左右
    │     - 高 > 宽：裁切上下
    │
    ├─ 5. 缩放到 256 × 256
    │
    ├─ 6. 压缩为 JPEG（质量 80%）
    │
    └─ 7. 写入 thumbnails/{mediaId}.jpg
```

---

## 3. 视频缩略图生成流程

```
输入：源视频文件路径
    │
    ├─ 1. 使用 MediaMetadataRetriever 提取视频某一帧
    │     - 默认提取第 1 秒（1,000,000 微秒）的帧
    │     - 如果视频时长 < 1 秒，提取第 0 帧
    │
    ├─ 2. 获取帧的 Bitmap
    │
    ├─ 3. 居中裁切为正方形（同图片流程）
    │
    ├─ 4. 缩放到 256 × 256
    │
    ├─ 5. 压缩为 JPEG（质量 80%）
    │
    └─ 6. 写入 thumbnails/{mediaId}.jpg
```

---

## 4. 缓存策略

| 规则 | 说明 |
|------|------|
| 缓存位置 | 应用私有目录/thumbnails/ |
| 缓存命中 | 文件存在且非空即为命中 |
| 缓存未命中 | 重新生成缩略图 |
| 缓存失效 | 缩略图文件被删除或损坏 |
| 缓存清理 | 删除媒体时同步删除对应缩略图文件 |
| 不使用内存缓存 | 缩略图直接从文件系统读取，依赖系统文件缓存 |

---

## 5. 默认占位图

当缩略图生成失败时，使用默认占位图：

| 场景 | 占位图 |
|------|--------|
| 图片加载失败 | 灰色背景 + 图片破损图标 |
| 视频加载失败 | 灰色背景 + 视频图标 |
| 缩略图生成异常 | 灰色背景 + 通用错误图标 |

**占位图实现方式**：
- 使用应用内嵌的 drawable 资源
- 不依赖网络加载
- thumbnailPath 字段存储占位图的资源标识符（非文件路径）

---

## 6. 性能规则

| 规则 | 说明 |
|------|------|
| 后台线程 | 所有缩略图生成在后台线程执行 |
| 逐个生成 | 批量导入时逐个生成，避免同时加载多张大图导致 OOM |
| 采样率计算 | 先读取图片尺寸再计算采样率，避免完整解码 |
| 超时处理 | 单个缩略图生成超过 10 秒，标记为失败并使用占位图 |
| 内存限制 | 生成过程中及时回收中间 Bitmap |

---

## 7. 缩略图删除规则

| 触发条件 | 操作 |
|---------|------|
| 删除单个媒体 | 同步删除 thumbnails/{mediaId}.jpg |
| 批量删除媒体 | 在事务中批量删除所有关联缩略图文件 |
| 清除全部数据 | 删除整个 thumbnails/ 目录 |

---

## 8. 验证标准

完成本 skill 后，必须满足以下全部条件：

- [ ] 图片缩略图正确生成（256×256 JPEG）
- [ ] 视频缩略图正确提取（第 1 秒帧）
- [ ] 居中裁切逻辑正确
- [ ] 采样率计算正确（大图不 OOM）
- [ ] 缩略图文件正确存储在 thumbnails/ 目录
- [ ] 生成失败时使用默认占位图
- [ ] 删除媒体时缩略图同步删除
- [ ] 生成在后台线程执行
- [ ] 缩略图生成器通过 DI 正确提供
