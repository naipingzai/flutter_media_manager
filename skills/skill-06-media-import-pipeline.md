# Skill-06: 媒体导入管线规范

## 前置依赖
skill-01, skill-05

## 目标
定义从外部存储导入媒体文件到应用私有目录的完整处理流程。

---

## 1. 导入管线概述

导入操作分为 6 个步骤，每步必须按顺序执行：

```
Step 1: 验证选择 → Step 2: 计算哈希 → Step 3: 检查重复
→ Step 4: 复制文件 → Step 5: 提取元数据 → Step 6: 生成缩略图
→ 保存到数据库
```

---

## 2. Step 1: 验证选择

| 验证项 | 规则 | 失败处理 |
|--------|------|---------|
| 文件列表非空 | 用户必须选择至少 1 个文件 | 显示"请选择要导入的文件"提示 |
| 文件可读 | 每个文件必须可读 | 跳过不可读文件，记录到失败列表 |
| 文件类型有效 | MIME type 必须是支持的图片或视频类型 | 跳过不支持的文件，记录到失败列表 |
| 存储空间充足 | 应用私有目录剩余空间 ≥ 所选文件总大小 × 1.1 | 显示"存储空间不足"提示 |

---

## 3. Step 2: 计算文件哈希

| 规则 | 说明 |
|------|------|
| 算法 | SHA-256 |
| 计算范围 | 整个文件内容 |
| 计算时机 | 复制文件之前 |
| 并发策略 | 逐个计算（避免内存溢出） |
| 失败处理 | 计算失败的文件跳过，记录到失败列表 |

---

## 4. Step 3: 检查重复

| 规则 | 说明 |
|------|------|
| 去重依据 | SHA-256 哈希值 |
| 检查方式 | 查询数据库中是否已存在相同 hash 的记录 |
| 重复处理 | 跳过该文件，记录到"已存在"列表 |
| 提示方式 | 导入完成后显示"N 个文件已存在，已跳过" |

---

## 5. Step 4: 复制文件

| 规则 | 说明 |
|------|------|
| 源路径 | 外部存储中的原始文件路径 |
| 目标路径 | 应用私有存储目录 |
| 文件命名 | 使用原始文件名，如果已存在同名文件则添加数字后缀（如 IMG_001(2).jpg） |
| 复制方式 | 流式复制（逐块读写），避免一次性加载整个文件到内存 |
| 进度报告 | 每复制完一个文件，更新进度（已处理/总数） |
| 失败处理 | 复制失败的文件跳过，清理已复制的部分文件，记录到失败列表 |
| 原始文件 | 不删除原始文件（只读导入） |

---

## 6. Step 5: 提取元数据

对每个成功复制的文件，提取以下元数据：

| 元数据 | 来源 | 说明 |
|--------|------|------|
| 文件名 | 文件系统 | 原始文件名 |
| 文件大小 | 文件系统 | 字节数 |
| MIME type | 文件系统 / 内容检测 | 如 "image/jpeg" |
| 宽度 | 图片/视频元数据 | 像素 |
| 高度 | 图片/视频元数据 | 像素 |
| 时长 | 视频元数据 | 毫秒，仅视频 |
| 创建时间 | 导入时间 | 使用当前时间，不读取 EXIF |

**提取失败处理**：
- 宽度/高度无法提取 → 设为 null
- 时长无法提取（视频） → 设为 null
- MIME type 无法检测 → 使用文件扩展名推断

---

## 7. Step 6: 生成缩略图

缩略图生成参见 skill-07 的完整规范。

| 规则 | 说明 |
|------|------|
| 生成时机 | 文件复制完成后立即生成 |
| 失败处理 | 生成失败时使用默认占位图路径，thumbnailPath 不允许为 null |

---

## 8. 保存到数据库

所有元数据提取和缩略图生成完成后，在数据库事务中执行：

| 步骤 | 操作 |
|------|------|
| 1 | 生成 UUID |
| 2 | 构建 MediaItem 实体 |
| 3 | 插入数据库 |

**批量导入事务规则**：
- 单个文件：单独插入
- 多个文件：批量插入（在事务中执行，一条失败全部回滚）

---

## 9. 进度报告规范

导入过程中需要向 UI 层报告进度：

| 阶段 | 进度信息 | 说明 |
|------|---------|------|
| 开始 | total = 文件总数 | 初始化进度条 |
| 处理中 | current = 已处理数, fileName = 当前文件名 | 更新进度条和文件名 |
| 完成 | successCount, skippedCount, failedCount, failedFiles | 显示导入结果 |
| 取消 | 已导入的文件保留，未处理的跳过 | 用户可随时取消 |

---

## 10. 导入结果报告

导入完成后显示结果摘要：

| 字段 | 说明 |
|------|------|
| 成功数 | 成功导入的文件数量 |
| 跳过数 | 因重复跳过的文件数量 |
| 失败数 | 因错误失败的文件数量 |
| 失败文件列表 | 每个失败文件的文件名和失败原因 |

**结果展示形式**：
- 如果全部成功且无跳过：显示"已导入 N 个文件"Toast
- 如果有跳过或失败：显示结果对话框，列出详细信息

---

## 11. 取消导入

| 规则 | 说明 |
|------|------|
| 取消时机 | 用户可随时点击"取消"按钮 |
| 取消行为 | 停止处理剩余文件 |
| 已导入文件 | 保留在数据库中，不回滚 |
| 正在复制的文件 | 完成当前文件的复制后停止 |
| 清理 | 删除已复制但未保存到数据库的临时文件 |

---

## 12. 导入触发方式

| 触发方式 | 说明 |
|---------|------|
| FAB → 从设备导入 | 在首页点击 FAB 菜单中的"从设备导入"选项 |
| FAB → 选择目录导入 | 在首页点击 FAB 菜单中的"选择目录导入"选项 |

---

## 13. 验证标准

完成本 skill 后，必须满足以下全部条件：

- [ ] 6 步管线全部实现，无省略
- [ ] SHA-256 去重正确工作
- [ ] 重复文件被正确跳过并记录
- [ ] 文件正确复制到应用私有目录
- [ ] 文件名冲突时正确添加后缀
- [ ] 元数据正确提取（宽高、时长、MIME type）
- [ ] 缩略图正确生成（参见 skill-07）
- [ ] 进度正确报告到 UI 层
- [ ] 导入结果正确汇总和展示
- [ ] 取消操作正确执行，已导入文件保留
- [ ] 存储空间不足时正确提示
- [ ] 全部失败时有明确的错误提示
